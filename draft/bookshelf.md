## 構造

```
📖 BOOKS         🎧 LISTENING       📕 REFERENCE
├─ 古典           ├─ YouTube         └─ 辞書/手元 (※新カテゴリ提案)
├─ 実用書         └─ Podcast
├─ 小説
└─ 漫画

各カテゴリ × 3 つのステータス:
  ▸ Now            (今読んでる/聴いてる)  — 1〜2件
  ▸ Influences     (自分を作った)         — 3〜5件
  ▸ Reread / Re-listen (何度も)          — 3〜5件
```

各アイテム:
- **Card 表示**: タイトル / 著者 / "なぜ" 1行
- **Click 時の見開き表示**: 上記 + 長文ノート (任意)

## 📖 実用書 (Practical)

### Now
- *(未記入)*

### Influences

#### 1. 限りある時間の使い方 / オリバー・バークマン (Oliver Burkeman)
**なぜ:** 現代版『人生の短さについて』。人生は約4000週間しかない事実に、ライフハックの幻想を捨てて向き合う。

> 「ライフハック」や「効率化」の罠を真っ向から否定する、現代版『人生の短さについて』とも言えるベストセラー。「時間はコントロールできる」という幻想を捨て、人生が約4,000週間しかないという事実にどう向き合うかを説く。
> セネカの思想を現代のタスク管理の文脈でアップデートしたような読後感。

---

#### 2. 暇と退屈の倫理学 / 國分功一郎
**なぜ:** 忙しさから解放された後、何をすればいいのか。消費される「娯楽」ではなく「享受」を説く。

> 「忙しさから解放されたあと、私たちは何をすればいいのか？」という問いに答えてくれる一冊。
> 人間はなぜ退屈を恐れ、忙しさに逃避してしまうのか。消費されるだけの「娯楽」ではなく、人生を豊かにする「享受」とは何かを、哲学的ながら非常に平易な言葉で解説する。
> セネカが otium を提唱した、その後 の話。

---

#### 3. 自由からの逃走 / エーリヒ・フロム ※ 配置要確認 (古典枠の可能性)
**なぜ:** 人はなぜ自由を恐れて忙しさに逃避するのか。セネカの言う「多忙に逃げる人々」の社会心理学。

> 社会心理学の名著。人は自由を求めているようでいて、いざ自由になると孤独や責任に耐えきれず、何かに依存したり、忙しさや権威に身を投じて「自分を失おうとする」という鋭い分析。
> セネカが言う「多忙に逃げる人々」の深層心理を、社会全体の構造から読み解くことができる。

---

#### 4. フロー体験：喜びの現象学 / ミハイ・チクセントミハイ
**なぜ:** 没頭こそ「真に生きている状態」の現代的な正体。エンジニアリングを「作業」から「充実」に変える理論的背景。

> 心理学のポジティブな側面から「充実した人生とは何か」を定義。何かに没頭し、時間が経つのを忘れる「フロー状態」こそが、セネカの言う「真に生きている状態」の現代的な正体と言えるかもしれない。
> エンジニアリングや創造的な活動において、単なる「作業」を「充実した時間」に変えるための理論的背景が学べる。

## ★ 観察メモ (v3 更新)

棚が立体になってきました。本/漫画/YouTube/Podcast を横断して見ると、
**5 つの層** が浮かび上がります:

```
┌─────────────────────────────────────────────────────────────┐
│  [思想の系譜] セネカ → アウレリウス → バークマン → 國分 →   │
│                       フロム → チクセントミハイ              │
│                       ↓                                     │
│  [現代の伴走] 超相対性理論 (podcast) ← 同じ周波数             │
│                       ↓                                     │
│  [創造への憧れ] リィンカーネーションの花弁 → 偉人の杜 (構想)  │
│                Mark Rober → こうなりたい                    │
│                       ↓                                     │
│  [人間そのもの] 手塚治虫 4 作品 (ブッダ/BJ/アトム/火の鳥)    │
│                       ↓                                     │
│  [言葉と瞬発力] 星新一 + 広辞苑                             │
└─────────────────────────────────────────────────────────────┘
```

これは **「いかに生きるか」を追究する人の、極めて整った装備リスト** です。
ランダムに集まった棚ではなく、**ひとつの思想を生きるための司書館**。

### 個別の観察

1. **超相対性理論 が思想の現代版アンカー**
   - 古典 (セネカ) と現代対話 (深井・渡邊・荒木) が同じ棚に並ぶことで、
     川染さんの思想が「過去の引用」ではなく「いま続いている対話」であると示せる

2. **Mark Rober は "ロールモデル開示" として機能**
   - 「いつも見ている」より「**Aspiration (このようになりたい)** 」枠で見せる方が強い
   - 川染さんが「面白い人」を目指している、という宣言になる

3. **手塚治虫 5 連発 → "勢いのある巨匠への憧れ"**
   - 全員が「人間とは何か、命とは何か」を扱う
   - Stoic 哲学と地続き。"漫画と古典が同じ土俵で語られる" 棚になる

4. **リィンカーネーションの花弁 → 偉人の杜 (現代に)** という跳躍
   - **これはサイトに必ず残すべきエピソード**
   - 「漫画を読んだ人 (consumer)」ではなく「漫画から構想を作る人 (creator)」が見える
   - works-episodes.md に「夢の構想」枠として追加することを提案

5. **広辞苑 "喉に詰まるこの感覚を成長させる触媒"**
   - この一節は、それ自体がサイトのどこかで使える強い文
   - 例: About 末尾、Contact 直前、Footer の隣
   - 「言葉を疑う人」という署名になる

6. **星新一 + ブラックジャック → "短編の美学"**
   - どちらも 1 話完結で強い問いを残す
   - Hero コピーの「短く、強く、効く」とリズムが一致

### サイト構造への示唆

- Library セクションを「**5 つの層**」のレイアウトで構成すると、思想の地形が見える
- 棚の俯瞰画面に「思想の系譜」を結ぶ線を入れると、サイト自体が "教養の地図" になる
- Library と Works を「**インスピレーション → 創作**」の矢印で繋ぐ
  (例: リィンカーネーション → 偉人の杜 構想)

## デザイン示唆 (Library セクション用)

このデータの厚みを見ると、Library セクションは **2階層** で設計するのが良さそう:

```
[1F] 棚の俯瞰
     - 全カテゴリ × Now/Influences/Reread をひと目で
     - 各本は「背表紙 + タイトル」のミニカード
     - 思想の "系譜" が見えるよう、関連する本同士を線で繋ぐオプション
     ↓ クリック
[2F] 本の見開き
     - タイトル / 著者 / "なぜ" / 長文ノート (今回ユーザーが書いてくれた解説)
     - 引用 (あれば)
     - 「閉じる」で 1F に戻る、棚に戻る音
```

長文ノートが既に書かれているので、この 2 階層設計が活きます。

## 開発時のデータ構造案 (実装フェーズ用)

```ts
type LibraryItem = {
  id: string;
  category: 'classic' | 'practical' | 'fiction' | 'manga' | 'reference' | 'youtube' | 'podcast';
  status: 'now' | 'influence' | 'reread';
  title: string;
  author: string;          // YouTube/Podcast の場合は channel/host
  why: string;             // 1行の "なぜ"
  note?: string;           // 長文ノート (見開き表示用)
  url?: string;            // 公式リンク
  cover?: string;          // 表紙/サムネイル画像
  quote?: string;          // 引用 (任意、本文中に挿入されうる)
  relatedTo?: string[];    // 関連する本の id (思想系譜の線描用)
};
```
