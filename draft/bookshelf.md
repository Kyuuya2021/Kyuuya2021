## 📖 実用書 (Practical)

- *(未記入)*

### Influences

#### 1. 限りある時間の使い方 / オリバー・バークマン (Oliver Burkeman)
**なぜ:** 現代版『人生の短さについて』。人生は約4000週間しかない事実に、ライフハックの幻想を捨てて向き合う。

> 「ライフハック」や「効率化」の罠を真っ向から否定する、現代版『人生の短さについて』とも言えるベストセラー。「時間はコントロールできる」という幻想を捨て、人生が約4,000週間しかないという事実にどう向き合うかを説く。
> セネカの思想を現代のタスク管理の文脈でアップデートしたような読後感。

---

#### 2. 暇と退屈の倫理学 / 國分功一郎
**なぜ:** 忙しさから解放された後、何をすればいいのか。消費される「娯楽」ではなく「享受」を説く。

> 「忙しさから解放されたあと、私たちは何をすればいいのか？」という問いに答えてくれる一冊。
> 人間はなぜ退屈を恐れ、忙しさに逃避してしまうのか。消費されるだけの「娯楽」ではなく、人生を豊かにする「享受」とは何かを、哲学的ながら非常に平易な言葉で解説する。
> セネカが otium を提唱した、その後 の話。

---

#### 3. 自由からの逃走 / エーリヒ・フロム ※ 配置要確認 (古典枠の可能性)
**なぜ:** 人はなぜ自由を恐れて忙しさに逃避するのか。セネカの言う「多忙に逃げる人々」の社会心理学。

> 社会心理学の名著。人は自由を求めているようでいて、いざ自由になると孤独や責任に耐えきれず、何かに依存したり、忙しさや権威に身を投じて「自分を失おうとする」という鋭い分析。
> セネカが言う「多忙に逃げる人々」の深層心理を、社会全体の構造から読み解くことができる。

---

#### 4. フロー体験：喜びの現象学 / ミハイ・チクセントミハイ
**なぜ:** 没頭こそ「真に生きている状態」の現代的な正体。エンジニアリングを「作業」から「充実」に変える理論的背景。

> 心理学のポジティブな側面から「充実した人生とは何か」を定義。何かに没頭し、時間が経つのを忘れる「フロー状態」こそが、セネカの言う「真に生きている状態」の現代的な正体と言えるかもしれない。
> エンジニアリングや創造的な活動において、単なる「作業」を「充実した時間」に変えるための理論的背景が学べる。

## ★ 観察メモ

第1ラウンドで集まった本 (5冊 + 1辞書 + SF) を見渡して気づいたこと:

1. **すべて「いかに生きるか」で繋がっている**
   - セネカ (古典) → アウレリウス (古典) → バークマン (現代翻訳) → 國分 (現代哲学) → フロム (社会心理) → チクセントミハイ (心理学)
   - 偶然ではなく、明確に同じ問いを違う角度から掘っている
   - **→ Library セクションは "ランダムな趣味の棚" ではなく "思想の系譜" として並べるべき**

2. **広辞苑が独特**
   - 言葉に対する敬意。テック起業家としては珍しい
   - サイト内で「言葉を選ぶ人」という側面の証拠になる

3. **星新一 = 瞬発力 + アイデア勝負**
   - エンジニア/起業家の感性と直結
   - 「短くて強い」は Hero コピーと共鳴する

4. **漫画がまだない**
   - ここに 1〜3 冊入ると、サイト全体の温度が一段上がる
   - 「セネカと並んで漫画」というギャップが効く
   - 候補質問: 中高生のとき何度も読んだ漫画？ いま手元にある漫画？

## デザイン示唆 (Library セクション用)

このデータの厚みを見ると、Library セクションは **2階層** で設計するのが良さそう:

```
[1F] 棚の俯瞰
     - 全カテゴリ × Now/Influences/Reread をひと目で
     - 各本は「背表紙 + タイトル」のミニカード
     - 思想の "系譜" が見えるよう、関連する本同士を線で繋ぐオプション
     ↓ クリック
[2F] 本の見開き
     - タイトル / 著者 / "なぜ" / 長文ノート (今回ユーザーが書いてくれた解説)
     - 引用 (あれば)
     - 「閉じる」で 1F に戻る、棚に戻る音
```

長文ノートが既に書かれているので、この 2 階層設計が活きます。

## 開発時のデータ構造案 (実装フェーズ用)

```ts
type LibraryItem = {
  id: string;
  category: 'classic' | 'practical' | 'fiction' | 'manga' | 'reference' | 'youtube' | 'podcast';
  status: 'now' | 'influence' | 'reread';
  title: string;
  author: string;          // YouTube/Podcast の場合は channel/host
  why: string;             // 1行の "なぜ"
  note?: string;           // 長文ノート (見開き表示用)
  url?: string;            // 公式リンク
  cover?: string;          // 表紙/サムネイル画像
  quote?: string;          // 引用 (任意、本文中に挿入されうる)
  relatedTo?: string[];    // 関連する本の id (思想系譜の線描用)
};
```

## 次に書き足してほしい

優先度順:

1. **星新一 と 広辞苑 の "なぜ" 1行** ← 既にある本のメタデータを完成させる
2. **漫画 1〜3 冊** ← 思想と肩書きから少し離れる、人柄の温度を入れる
3. **YouTube 1〜3 チャンネル** ← 思考の食事、現在進行形の自分が見える
4. **Podcast 1〜3 番組** ← 同上
5. **Now (今読んでる本)** ← 各カテゴリで 1冊ずつあれば理想
6. **広辞苑の置き場所** ← Reference カテゴリ採用してOK？ 古典の隅でも可
7. **自由からの逃走の置き場所** ← 実用 / 古典 のどちら扱い？

→ 1〜3 だけでも先に投げてくれれば、棚の "形" がほぼ完成します。
