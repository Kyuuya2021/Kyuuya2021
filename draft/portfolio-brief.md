## 1. このサイトのゴール

> **「面白い人だな、一緒に何かやってみたい」と訪問者に思わせる。**

副次的に:
- 自分が「何をやっている人なのか」を、初対面の人に1〜2分で理解させる。
- 受賞や所属の権威に **頼らず**、しかし **持っている**。
- バラバラに見える活動が、内側で繋がっていることを伝える。

## 3. 中核仮説 — 位置づけ

訪問者が真っ先に抱く疑問は、おそらくこれです:

> **「電気工学の学生がなぜ画商をやっていて、Stanford で研究して、AI で農地を管理しているのか？」**

この **「なぜ全部やっているのか」** にサイト全体で答えるのが本筋。
受賞リストは答えではなく、答えを支える証拠です。

### 仮の答え（コピーの種）
**「ぜんぶ、同じ問いから始まっている。」**

中心の問い (案): 
- *「人間を前向きにするものは、何でできているか。」*
- *「美しさと、実装と、社会は、ひとつの円のなかにある。」*

→ ここを川染さん本人の言葉で固めるのが最重要。
**Q1.** 自分の中で、すべての活動を貫いている「問い」を1文で言うとしたら？

## 4. 「面白い」を出すための装置

| 装置 | 内容 | サイト上の現れ方 |
|---|---|---|
| **矛盾の同居** | 画商 ↔ ハードウェア / 茨城 ↔ Stanford / 学生 ↔ CTO | Hero で「肩書きを並べる」のではなく「矛盾を並べる」 |

| **未完成を見せる** | 進行中・失敗・迷いを開示 | 「Now」/「Field Notes」セクション |

## 5. Voice プロファイル

**3語**: *好奇心の濃い / 軽やかに大胆 / 余白がある*

### Do
- 動詞で書く（「〜している」「〜にいる」）
- 固有名詞を出す（「ルワンダの」「茨城高専の」「d.school の廊下で」）
- 一人称で書く（「私たち」より「自分は」「川染は」）
- 1段落 3 文以内
- たまに、句点で止める。短く。

### Don't
- 「最先端」「次世代」「イノベーション」「DX」（中身が消える単語）
- 受賞リストを文中に詰め込む（年表が別にある）
- 「〜させていただいております」（サイトでは距離を取りすぎる）
- すべてを完成形で語る（伸びしろが消える）

### サンプル文（短）
- 🔴 NG: 「最先端の AI 技術を駆使し、革新的なソリューションを提供しています。」
- 🟢 OK: 「ルワンダのコーヒー農家のために、AI で農地を見張る仕組みを作っている。3 年前、JICA の人に呼ばれて現地に飛んだのが始まりだった。」

## 6. Hero コピー候補（5案）

> どれも 2 行構成（**Hook** → **Identity**）。実装ではタイピング・モーフィング等で
> 1 行目を変化させる演出が想定。

### A. 矛盾並列型
```
画商で、ドローンを作っている。
茨城のガレージから、Stanford で、ルワンダで。
— 川染 有哉 / Student Entrepreneur × Researcher
```
**狙い**: 「は？」と立ち止まらせる。記憶に残る。
**リスク**: ふざけて見える可能性。

### B. 問い先行型 *(◎ 推奨)*
```
人を前向きにするものは、何でできているか。
それを確かめるために、画商も、AIも、研究もやっている。
```
**狙い**: ビジョン「誰もが前向きに〜」と直結。哲学者っぽい温度。
**リスク**: 抽象度が高い。

### C. 場所と時間型
```
2026年4月、茨城のつくばで、Stanford 経由の研究員が、
画商の朝とコードの夜のあいだで動いている。
```
**狙い**: 映画的。日常と肩書きを地続きにする。
**リスク**: 詩的すぎて読み飛ばされる可能性。

### D. 革命家ストレート型
```
新しい職業をひとつ、作っている。名前は「革命家」。
電気工学から始まった人間が、なぜここに辿り着いたのか。
```
**狙い**: ビジョンを正面から。ストーリー導線が立つ。
**リスク**: 言葉が重い。本人の身体感覚に合うか要確認。

### E. 動詞だけ型
```
作る。売る。研究する。育てる。組む。賭ける。
それでもまだ、足りない。
```
**狙い**: 多動性を音で見せる。スクロール開始のリズム。
**リスク**: 文脈なしには空疎。下に続く About への接続が必須。

> **Q2.** A〜E のどれが「自分の声」に近いですか？ 全部違うなら、どこが違う？
> **Q3.** 一行目の英訳バージョン (バイリンガル切替時) は、直訳 / 意訳 どちらが好み？

## 7. About 草稿 (300字版)

> ここはまだ仮。声が決まったら書き直します。

```
電気工学から始まった。自動運転EVとドローンを作っていた高専生が、
ある日、絵を売り始めた。日本の画家を守りたかった。
同じ年、ルワンダにいた。コーヒー農家のために AI を作りに行った。

肩書きが増えるたびに、「結局なにがやりたいんですか」と聞かれる。
答えはひとつ。「人が前向きに生きていける社会を、ひとつ作りたい」。
そのために、いまは Stanford で研究をして、株式会社プライアの CTO をして、
画商をやって、AI を組んで、ピッチで勝って、また現場に戻る。

革命家、という職業を、自分で作っているところ。
```

## 8. Works のエピソード設計

各プロジェクトに以下の3点をワンセットで持たせる:

```
[1] きっかけ (Why) — 1 行
    例: 「JICA の人に呼ばれて、ルワンダ行きの飛行機に乗った。」

[2] やっていること (What) — 2-3 行
    例: 「コーヒー農地を衛星画像と地上センサーで見張り、品質と
       収穫タイミングを農家に届ける AI を作っている。
       現地で6回、検証を回した。」

[3] いま何が面白いか (Now) — 1 行
    例: 「収穫量より、農家の意思決定の質を変える方が効くと気づいた。」
```

> (画商 / コーヒー×AI / EV・ドローン / Stanford d.school / プライア / その他)

## 9. Interactive Seeds — サイト自体を作品にするための種

> 全部やる必要はないです。**1〜2 個に絞って深く** が良い。

| # | 種 | 概要 | 実装の重さ |
|---|---|---|---|
| 1 | **問いの円** | 中心に「問い」、周囲に Art / Tech / Ent / Research の4軸が浮かぶ。ドラッグで角度を変えると、4軸が交わる場所に Works が浮かび上がる。 | 中 (Canvas / SVG + Framer Motion) |
| 2 | **筆跡カーソル** | カーソルが筆になっていて、サイトに墨の跡が残る。一定時間で消える。 | 軽 (Canvas) |
| 3 | **時間軸スクラブ** | 画面下に 2018 → 2026 のスライダー。動かすと Works が時系列で現れ消える。 | 中 |
| 4 | **Hero 生成画** | アクセスのたびに違う「線画 × 文字組」を生成してヘッダーにする。p5.js / WebGL シェーダ。 | 重 |
| 5 | **回路図モチーフ** | 電気工学のバックグラウンドを活かして、各セクションの間を回路図的な線で繋ぐ。スクロールすると電流が流れる。 | 中〜重 |
| 6 | **画商ギャラリー** | Art セクションだけ別の質感に切り替わる (白背景、明朝、間)。サイト内で美術館に入る体験。 | 軽〜中 |
| 7 | **コマンドラインの自分** | サイト下部に常駐するミニ CLI。`whoami` `now` `stanford` 等のコマンドで川染さんの「現在地」が返る。 | 軽 |

**Q6.** ピンと来た 1〜2 個はどれ？ 「これは違う」というのも教えてください。

## 11. 進め方の提案

```
Phase 1 (いま) — 位置づけと声を固める ← このドキュメント
  ↓
Phase 2 — ChatGPT で画像生成 / ムード探索
  - 第10節のブリーフをベースに 3-5 ラウンド回す
  - 出てきた方向性を「色 / タイポ / モーション」のトークンに翻訳
  ↓
Phase 3 — Claude.ai (artifacts) でデザイン実装
  - Hero と About を最優先で作る (1 ページの 30%)
  - 他セクションは後回しで OK
  ↓
Phase 4 — Cursor + Claude Code CLI で本実装
  - Phase 3 のコードを Next.js に移植
  - i18n / SEO / メディア最適化
  - データ構造 (受賞 / Works) はこの段階で投入
  ↓
Phase 5 — 公開後の運用
  - Now / Field Notes セクションを月次更新
  - Zenn 連携 (RSS) を起動
  - 1 年に 1 回、Hero の生成画を更新
```
